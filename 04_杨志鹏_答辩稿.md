# 杨志鹏答辩稿

各位老师好，我是杨志鹏，我主要负责本项目中的“研究空白与前沿探测”模块。

这个模块的目标是帮助用户在大量文献中快速看出一个研究方向的发展趋势、研究热点和潜在空白。传统文献阅读往往依赖人工判断，效率比较低，而且容易受到个人经验影响。我们希望通过数据分析的方法，把文献中的年份、关键词、主题和研究方法等信息进行统计，让用户更直观地看到这个领域的发展情况。

我的模块主要使用 Python 和 Pandas 完成数据处理。首先，我会接收文献检索模块返回的标准化数据，包括标题、摘要、年份、关键词、来源数据库等字段。然后对这些数据进行清洗，例如去除空值、统一年份格式、合并相近关键词等。清洗完成后，再进行统计分析。

在热点分析方面，我主要从三个角度入手。第一是关键词频次统计，找出出现次数较高的研究主题。第二是时间趋势分析，观察某些关键词在不同年份的增长情况，从而判断哪些方向正在升温。第三是主题分布分析，把文献按照研究方法、应用场景或疾病类型进行分类，帮助用户理解领域结构。

在研究空白识别方面，我们不是简单地找出现次数少的词，而是结合“低覆盖”和“高潜力”两个思路进行判断。例如某个方向近几年增长较快，但相关文献总量还不多，可能说明它是一个新兴前沿方向；又比如很多论文都提到某个问题尚未解决，那么系统就会把它作为潜在研究空白推荐给用户。

本模块最终会输出三类结果：第一类是当前研究热点，例如高频关键词和热门方法；第二类是趋势变化，例如某些主题在近几年快速增长；第三类是研究空白建议，例如可以继续深入探索的科学问题。这些结果会提供给前端进行图表展示，也会被写入最终报告。

开发中比较重要的问题是如何让分析结果具有可解释性。如果系统只给出一句“这是热点”，用户很难相信。所以我的设计是让每个结论尽量附带数据依据，例如相关文献数量、年份变化趋势、关键词频次等。这样老师和用户在查看结果时，可以知道结论是从哪里来的。

通过这个模块，我们把文献检索结果进一步转化为了科研决策依据，为后续实验方案设计提供方向支撑。我的汇报到这里，谢谢各位老师。
