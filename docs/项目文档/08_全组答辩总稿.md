# 全组答辩总稿

各位老师好，我们小组本次实习最终项目的题目是“科学文献研究智能体：从综述到实验设计”。本项目面向科研人员、研究生和企业研发团队，目标是帮助用户更高效地完成文献检索、综述整理、研究热点分析、研究空白识别、实验方案设计和论文写作辅助等工作。

我们选择这个题目的原因是，在真实科研过程中，前期文献调研和实验设计往往需要消耗大量时间。用户不仅要在多个数据库中检索文献，还要判断研究热点、发现潜在创新点，并进一步设计实验方案。我们的项目希望利用人工智能技术，把这些环节串联成一个完整的智能科研辅助流程。

本项目采用前后端分离架构。后端使用 Python 和 Django 进行开发，负责业务接口、用户权限、系统日志和数据库交互；前端使用 Vue.js 进行开发，负责页面展示、图表可视化和报告预览；数据库使用 MySQL 保存用户信息、文献记录、分析结果和报告数据；智能体部分使用 LangChain 和 Agent 技术完成文献摘要、研究空白分析、实验方案生成和论文草稿生成；数据分析部分使用 Pandas 对文献数据进行清洗、统计和趋势分析。

我们的系统主要包括六个核心功能模块。

第一个模块是跨库文献智能检索与摘要。用户输入研究主题后，系统可以从 PubMed、arXiv 等学术数据库中进行跨库检索，并对检索结果进行整理、去重和摘要生成。系统最终会输出结构化文献综述，帮助用户快速了解该方向的研究背景、主要方法和代表性成果。

第二个模块是研究空白与前沿探测。系统会对已有文献进行统计分析，包括关键词频次、年份趋势和主题分布等内容，从而识别当前研究热点和前沿方向。同时，系统会结合文献覆盖情况和趋势变化，给出潜在研究空白建议，为后续研究提供方向参考。

第三个模块是实验方案设计建议。系统会根据用户选择的科学问题或研究空白，自动生成初步实验方案，包括研究目标、技术路线、实验步骤、方法推荐、预期结果和风险控制建议。这个模块的定位是为科研人员提供可修改的初稿，帮助用户更快形成实验设计思路。

第四个模块是论文写作辅助。系统可以根据文献综述、实验方案和用户笔记，生成论文引言、方法、结果、讨论和摘要等章节草稿，并对表达方式进行学术化处理，使内容更加符合论文写作规范。

第五个模块是用户权限与系统管理。系统区分管理员、运营人员和分析师等角色。管理员可以管理用户和系统日志，运营人员可以查看系统使用情况，分析师可以使用文献检索、分析和报告生成功能。系统日志会记录用户的关键操作，便于审计追踪。

第六个模块是可视化与报告生成。系统会通过图表展示文献数量趋势、关键词热度、研究方向分布和研究空白分析结果，并生成完整研究报告。报告内容包括研究主题、检索结果、文献综述、热点趋势、研究空白、实验方案和论文草稿，方便用户后续汇报和修改。

在团队分工方面，孟永琪作为组长，主要负责项目统筹、Django 后端架构、用户权限、系统日志和接口联调；郗纯瑜负责跨库文献检索与摘要模块；杨志鹏负责研究热点分析和研究空白探测模块；张智勇负责实验方案设计建议和论文写作辅助模块；王向前负责 Vue 前端页面、可视化展示、报告生成和 Docker 部署。

本项目的主要亮点有三个。第一，我们把科研前期流程进行了完整串联，从文献检索一直延伸到实验方案和论文草稿，不只是单点功能。第二，我们使用 Agent 技术对多个任务进行编排，使系统能够分步骤完成复杂科研辅助任务。第三，我们结合 Pandas 数据分析和前端可视化，让研究热点和空白识别过程更加直观、可解释。

通过本项目开发，我们进一步掌握了 Python、Django、Vue.js、MySQL、Pandas、LangChain 和 Agent 开发技术，也提升了团队协作、需求分析、接口设计、系统集成和答辩表达能力。以上就是我们小组项目的整体介绍，谢谢各位老师。
