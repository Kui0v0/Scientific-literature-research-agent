# 王向前答辩稿

各位老师好，我是王向前，我主要负责本项目中的“前端可视化与报告生成”模块，同时参与前后端联调和 Docker 部署。

这个项目虽然核心是智能体和数据分析，但如果没有清晰的前端页面，用户很难直观理解系统的分析结果。因此我负责把后端返回的文献数据、趋势分析、研究空白、实验方案和论文草稿，以更清楚的界面展示出来，并支持生成研究报告。

前端部分我们使用 Vue.js 开发，主要页面包括登录页面、研究主题输入页面、文献检索结果页面、趋势分析页面、实验方案页面、论文写作页面和报告预览页面。用户登录后，可以输入研究主题并发起检索，随后在不同页面查看系统生成的结果。

在可视化方面，我主要展示三类图表。第一类是文献数量趋势图，用来展示某个研究方向在不同年份的发文变化。第二类是关键词热度图，用来展示当前领域中出现频率较高的研究主题。第三类是研究方向分布图，用来帮助用户理解不同研究主题之间的比例关系。通过这些图表，用户可以更快判断一个方向是否热门、是否具有持续增长趋势。

在报告生成方面，我负责把各模块的输出整合到一个完整页面中，包括研究主题、检索条件、代表性文献、摘要综述、热点趋势、研究空白、实验方案和论文草稿。用户可以在报告页面进行预览，也可以导出为 Markdown、PDF 或 Word 格式，方便答辩展示和后续修改。

我参与的另一个工作是前后端联调。由于后端返回的数据来自多个模块，如果字段命名不统一，前端展示就会出错。因此我和组员一起确定了接口返回格式，例如文献列表、关键词统计、趋势数据和报告内容都采用统一的 JSON 结构。这样前端开发更稳定，也方便后期扩展。

在部署方面，我们考虑使用 Docker 和 Linux 环境进行部署。Docker 的好处是可以把后端、前端和数据库运行环境统一起来，减少因为不同电脑配置不同导致的运行问题。答辩演示时，也可以更方便地启动整个系统。

通过这个模块，系统不再只是后台生成结果，而是形成了一个用户能够实际操作、查看和导出的完整平台。我的介绍到这里，谢谢各位老师。
