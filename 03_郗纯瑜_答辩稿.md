# 郗纯瑜答辩稿

各位老师好，我是郗纯瑜，我主要负责本项目中的“跨库文献智能检索与摘要”模块。

在科研工作中，文献调研通常是第一步，也是最耗时间的一步。用户需要在多个数据库中反复检索、筛选和阅读大量文献。针对这个问题，我负责实现系统的文献获取和摘要整理能力，让用户输入一个研究主题后，系统能够自动从 PubMed、arXiv 等数据库中检索相关文献，并生成简明摘要和结构化综述。

我的模块主要包括四个部分。第一部分是检索参数设计。用户可以输入研究主题、关键词、时间范围和数据库来源，系统会把这些条件转换为对应数据库能够识别的检索请求。第二部分是跨库数据接入。不同数据库返回的数据格式不完全一样，比如标题、作者、摘要和发布时间字段名称可能不同，所以我需要把它们统一整理成系统内部的数据结构。

第三部分是文献去重和清洗。由于同一篇文章可能同时出现在多个数据库中，如果不处理会影响后续分析结果。因此我根据标题、DOI、发布时间等信息对文献进行去重，并清洗掉缺失严重或无效的数据。第四部分是智能摘要生成。系统会把检索到的文献摘要交给 LangChain Agent 进行整理，输出包括研究背景、主要方法、代表性成果和不足之处的结构化内容。

这个模块的技术重点是“跨库统一”和“摘要结构化”。如果只是简单展示搜索结果，用户仍然需要自己阅读大量内容；而我们的系统会进一步把文献内容压缩成可读性更强的综述结果。比如用户输入“单细胞测序在肿瘤免疫治疗中的应用”，系统会返回相关论文列表，并总结当前研究主要集中在哪些技术、哪些疾病场景和哪些分析方法上。

开发中遇到的难点是不同数据源返回内容差异较大。有的文献有完整摘要，有的只有标题和作者；有的数据库字段比较规范，有的需要二次处理。我的解决方法是先定义统一的文献数据模型，再为不同数据源编写适配逻辑，保证后续分析模块和前端展示模块拿到的数据格式一致。

通过这个模块，系统能够完成从“用户研究问题”到“可阅读文献综述”的第一步，为后续的热点分析、研究空白识别和实验设计提供基础数据。我负责的内容介绍完毕，谢谢各位老师。
