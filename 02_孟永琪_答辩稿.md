# 孟永琪（组长）答辩稿

各位老师好，我是本组组长孟永琪，主要负责项目统筹、系统架构设计、Django 后端开发、用户权限管理、系统日志以及各模块接口联调。

我们小组的项目题目是“科学文献研究智能体：从综述到实验设计”。选择这个题目的原因是，在真实科研过程中，文献检索、综述整理、研究方向判断和实验方案设计往往需要大量时间。我们希望利用 Python、LangChain、Agent、Django、Vue.js、Pandas 和 MySQL，搭建一个能够辅助科研人员完成前期研究工作的智能系统。

在项目整体设计上，我们采用前后端分离架构。前端使用 Vue.js 负责页面交互和结果展示，后端使用 Django 提供接口服务，MySQL 存储用户、文献、分析任务、生成结果和系统日志。智能体部分通过 LangChain Agent 进行任务编排，将“文献检索、摘要生成、热点分析、实验设计和论文写作”串成一条完整流程。

我负责的第一个重点是需求拆分和模块边界设计。我们把项目拆分为六个核心模块：跨库文献检索与摘要、研究空白与前沿探测、实验方案设计建议、论文写作辅助、用户权限与系统管理、可视化与报告生成。每个模块都有独立的输入输出格式，便于组员并行开发。

我负责的第二个重点是后端基础架构。后端主要包括用户模块、文献模块、分析模块、实验方案模块、写作模块和报告模块。为了便于系统维护，我在接口设计时尽量保持统一风格，例如任务创建接口、任务结果查询接口、报告导出接口都采用类似结构，这样前端联调时成本更低。

我负责的第三个重点是权限和日志。系统中设置管理员、运营人员和分析师等角色。管理员可以管理用户和系统日志，运营人员可以查看系统使用情况，分析师主要负责文献检索、分析和报告生成。系统日志会记录用户登录、发起检索、生成报告等关键行为，方便后期审计和问题定位。

在开发过程中，我们遇到的主要问题是不同模块之间数据格式容易不统一。例如检索模块返回的是文献列表，分析模块需要的是清洗后的字段，前端需要的是适合图表展示的数据。为了解决这个问题，我和组员一起约定了统一的数据字段，包括标题、作者、摘要、来源、年份、关键词、链接等，使各模块能够顺利衔接。

通过这个项目，我主要提升了后端项目组织、接口设计、团队协作和系统集成能力。这个系统的价值不只是简单生成一段回答，而是把科研前期工作拆成多个可执行步骤，并通过智能体把步骤连接起来。我的介绍到这里，谢谢各位老师。
